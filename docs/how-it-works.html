Aqueduct: how it works

Turning a data file into something an agent can query and pay for, per row.

Aqueduct separates a dataset's preparation from its use. Building happens once: a file is profiled, priced, checked, and sealed into a frozen configuration. Using happens on every request: an agent queries that configuration and pays per row over a payment channel. The two pictures below trace each path. Crucially, no language model runs on the request path — the configuration produced in Figure 1 is the exact, deterministic program executed in Figure 2.

The frozen tap from Figure 1 is exactly what the server runs in Figure 2.

Two separate jobs, one shared artifact. Building is a compile step that runs once; using is a fast, deterministic, pay-as-you-go path that runs again and again — and the publisher is paid for each answer.
